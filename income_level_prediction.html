Income Level Prediction - Data Science Capstone Project

Project Overview

This capstone project aims to build a machine learning model that predicts whether an individual's income exceeds $50,000 per year based on census data. By analyzing demographic and employment-related features, the goal is to gain insights into income distribution and to create a predictive tool that can support socio-economic research and policy planning.

The project follows a structured machine learning pipeline that includes:

  • Data Collection and Inspection
  • Exploratory Data Analysis (EDA)
  • Data Preprocessing and Feature Engineering
  • Model Selection, Training, and Evaluation
  • Insights and Recommendations

Timeline

Duration: 3 Weeks
Start Date: June 1st
End Date: June 2oth

Dataset

The dataset is derived from U.S. Census Bureau data and includes features such as age, education, work class, marital status, occupation, and capital gain/loss. The target variable is income, categorized as <=50K or >50K.

Tools & Technologies

  • Python
  • Pandas, NumPy
  • Matplotlib, Seaborn
  • Scikit-learn
  • Jupyter Notebook

Objective

To develop an accurate and interpretable machine learning model that predicts whether a person earns more than $50,000/year, and to extract actionable insights from the data.

Step 9: Summary and Recommendations

Summary

  • We explored a census income dataset with the goal of predicting whether an individual's income exceeds $50K per year.
  • During exploratory data analysis (EDA), we discovered meaningful insights:
    • Higher income levels are associated with advanced education, full-time work, and specific occupations.
    • Capital gains and hours worked per week show strong correlations with income.
  • Data preprocessing included:
    • Handling missing values using mode imputation.
    • Encoding categorical features using One-Hot Encoding.
    • Scaling numerical features and applying PCA for dimensionality reduction.
  • We trained and evaluated several machine learning models:
    • Logistic Regression
    • Random Forest
    • Support Vector Machine (SVM)
    • K-Nearest Neighbors (KNN)
    • Gradient Boosting
  • Gradient Boosting delivered the best performance across all key evaluation metrics.

Best Model Performance (Gradient Boosting)

  • Accuracy: ~87%
  • Precision: High (low false positive rate)
  • Recall: Strong (good at detecting true positives)
  • F1 Score: Balanced measure of precision and recall
  • ROC AUC: ~0.93, indicating excellent model discriminative ability

Recommendations

  1. Gradient Boosting is recommended for deployment due to its strong and consistent performance.
  2. For further optimization:
    • Apply hyperparameter tuning (e.g., using GridSearchCV or RandomizedSearchCV).
    • Explore ensemble strategies like stacking or blending multiple models.
    • Analyze feature importances to interpret key drivers of income.
  3. Deploy the best-performing model using a REST API (e.g., Flask or FastAPI).
  4. Continuously monitor model performance for signs of data drift or degradation.
  5. Consider gathering updated or more diverse data to enhance generalization and robustness across populations.
